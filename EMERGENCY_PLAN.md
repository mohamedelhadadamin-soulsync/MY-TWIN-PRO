# MyTwin – خطة الطوارئ (Emergency Plan)

## سيناريو 1: انهيار الخادم (Railway down)
1. تفعيل النسخة الاحتياطية على Railway Dashboard
2. مراقبة السجلات (Logs) لتحديد السبب
3. إعادة التشغيل التلقائي (restartPolicy: ON_FAILURE)

## سيناريو 2: تجاوز حدود API (Rate Limit)
1. تفعيل Redis للتخزين المؤقت
2. توزيع الأحمال بين المفاتيح الاحتياطية (لدينا 3 مفاتيح لكل مزود)
3. تقليل استهلاك المستخدمين المجانيين تلقائياً

## سيناريو 3: اختراق أمني
1. تفعيل جدار الحماية (WAF) على Railway
2. تدوير جميع مفاتيح API فوراً
3. إبلاغ المستخدمين المتأثرين
